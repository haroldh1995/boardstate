# BoardState UI Preservation Rules

## Hard Rule

Do not visually redesign BoardState unless explicitly instructed.

## Preserve

Always preserve:
- current background
- transparent / clear glass-style panels
- compact battlefield style
- compact permanent tiles
- current spacing philosophy
- current mobile-first feel
- current life tracker visual identity
- overlay-driven interaction model

## Allowed UI Additions

New features should appear as:
- options menu entries
- transparent overlays
- transparent modal panels
- action drawers
- floating widgets
- compact submenus
- subtle highlights
- temporary combat overlays

## Not Allowed

Do not add:
- desktop simulator windows
- giant visible stack panels
- always-visible opponent boards
- full shared battlefield views
- large permanent sidebars that crowd the board
- heavy visual borders
- theme replacements
- background replacements
- full layout redesigns

## Styling Rule

If new CSS is needed, add it carefully.

Do not replace global styling unless absolutely required.

The existing transparent-panel look is part of the app identity.
